# Operational Qualification (OQ) — Protocol template

1. **Scope** — Functions under test, linked URS requirements.
2. **Test environment** — Data set, user roles, time sync, audit on.
3. **Test cases** — Step, expected, actual, pass/fail, evidence link.
4. **Security** — Access denied where expected; e-sign challenge on controlled actions.
5. **Part 11** — Audit record immutability; session timeout; password policy.
6. **Deviations** — Track and disposition per quality procedure.
7. **Entry/exit criteria** — All critical tests pass; no open critical issues.
8. **Sign-off** — QA/Validation owner with dates.
